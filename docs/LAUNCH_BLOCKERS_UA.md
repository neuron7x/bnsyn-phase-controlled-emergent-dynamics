# Блокери готовності до запуску (UA)

Стан сформовано на основі детермінованих перевірок репозиторію.

## Поточні блокери

### B1 — Залежність `pyyaml` для gate-скриптів
- **Симптом:** `scripts/validate_pr_gates.py` та `scripts/validate_workflow_contracts.py` падали з `ModuleNotFoundError: yaml` у середовищі без test/dev-залежностей.
- **Вплив на запуск:** блокував локальну валідацію PR-gates і workflow-контрактів (release governance check).
- **Статус:** **закрито в цьому циклі** (додано явну перевірку залежності та детерміноване повідомлення про відсутню залежність).

## Послідовний план виконання (цикл за циклом)

### Цикл 1 — Виявлення блокерів (діагностика)
1. Прогнати release readiness.
2. Прогнати PR gates validation.
3. Зафіксувати точну помилку та її клас.

**Критерій завершення:** є конкретний список блокерів з відтворюваною командою.

### Цикл 2 — Усунення критичного блокера B1
1. Оновити `validate_pr_gates.py`:
   - відкладений імпорт YAML;
   - явний error class для відсутньої залежності;
   - стабільний fail-fast у `main()` з кодом завершення.
2. Оновити `validate_workflow_contracts.py` аналогічно.

**Критерій завершення:** скрипти повертають контрольовану помилку замість traceback при відсутньому `pyyaml`.

### Цикл 3 — Підтвердження готовності
1. Встановити `pyyaml` у runtime середовищі.
2. Прогнати тести для змінених валідаторів.
3. Повторно прогнати PR-gates validation.
4. Перевірити quickstart-контракт.

**Критерій завершення:** усі релевантні перевірки проходять; blocker B1 закрито.

## Підсумок готовності
- Критичних блокерів, що **зараз** блокують запуск, не виявлено після закриття B1.
- Рекомендовано фіксувати запуск через стандартну послідовність: readiness → pr-gates → quickstart contract.
